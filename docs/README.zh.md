## 💡 概述

**DeBHuB** 是全球首个完全由 Pure Irys L1 DataChain 驱动的完全去中心化 Backend-as-a-Service 平台。

**零后端。零数据库。纯区块链。**

无需传统后端服务器和数据库，仅使用区块链智能合约和永久存储即可提供所有后端功能。

---

## 🏗️ 架构

```
前端 (React + Vite)
    ↓ 直接 RPC 连接
Irys L1 DataChain (链 ID: 1270)
    ├─ 智能合约层 (6个合约)
    │  ├─ DocumentRegistry    - 文档注册与管理
    │  ├─ AccessControl        - 权限与访问控制
    │  ├─ ProvenanceChain      - 数据出处追踪
    │  ├─ EventBus             - 事件发布/订阅
    │  ├─ CacheController      - 缓存失效控制
    │  └─ SearchIndex          - 搜索索引管理
    └─ 永久存储 (Irys 原生)
        └─ 不可变永久数据存储
```

### 技术栈
- **前端**: React 18, Vite 5, TypeScript 5, TailwindCSS
- **区块链**: Irys L1 DataChain, ethers.js v6, wagmi v2
- **缓存**: IndexedDB (idb v8.0.0)
- **存储**: Irys 原生永久存储

## 🔄 用户流程

### 1. 钱包连接与网络设置
```
用户访问应用
    ↓
自动检测 IrysVM 网络
    ↓
[使用不同网络?]
    ├─ 是 → 自动切换提示 (2秒后自动切换)
    └─ 否 → 请求钱包连接
        ↓
    连接 MetaMask/WalletConnect
        ↓
    进入仪表板
```

### 2. 文档创建流程
```
开始创建文档
    ↓
1. 检查 IndexedDB 缓存 (5分钟 TTL)
    ├─ 缓存命中 → 立即加载
    └─ 缓存未命中 → 查询区块链
        ↓
2. 输入文档内容
    ↓
3. 调用 DocumentRegistry.registerDocument()
    ↓
4. 签署交易 (用户)
    ↓
5. 永久存储在区块链上
    ↓
6. 通过 EventBus 发出 DocumentCreated 事件
    ↓
7. 自动更新 SearchIndex
    ↓
8. 更新 IndexedDB 缓存
    ↓
文档注册完成
```

### 3. 文档检索流程
```
请求文档检索
    ↓
1. 检查 IndexedDB 缓存
    ├─ 缓存有效 (< 5分钟) → 从缓存返回
    └─ 缓存过期/不存在
        ↓
2. 通过 RPC 调用 DocumentRegistry.getDocument()
    ↓
3. 从区块链查询数据
    ↓
4. 通过 AccessControl 验证权限
    ↓
5. 返回数据并缓存到 IndexedDB
    ↓
显示文档
```

### 4. 权限管理流程
```
请求权限设置
    ↓
1. 调用 AccessControl.grantAccess()
    ↓
2. 验证所有者
    ↓
3. 设置权限级别 (READ/WRITE/ADMIN)
    ↓
4. 签署交易
    ↓
5. 在区块链上存储权限信息
    ↓
6. 通过 EventBus 发出 AccessGranted 事件
    ↓
7. 通过 CacheController 使相关缓存失效
    ↓
权限设置完成
```

## ✨ 核心功能

### 完全去中心化
- 无需后端服务器
- 无需数据库
- 所有逻辑均以智能合约实现
- 客户端直接向区块链发起 RPC 调用

### 基于智能合约的数据管理
- 6个 Solidity 合约提供所有后端功能
- 文档注册、权限管理、出处追踪、事件处理
- 直接在区块链上执行的可编程数据逻辑

### 永久存储
- 利用 Irys L1 的原生永久存储
- 保证数据不可变性
- 数据所有权完全归属用户

### 高性能缓存
- IndexedDB 客户端缓存
- 5分钟 TTL 实现快速数据访问
- 最小化网络负载

### 自动网络切换
- 自动检测 IrysVM 网络
- 使用不同网络时自动切换提示
- 提供无缝用户体验
